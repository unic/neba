Download NEBA: Choose your package

Installation in Adobe AEM™

The following ZIP file is a CRX package containing both NEBA and the runtime dependencies not provided by Sling, e.g. Spring and Gemini Blueprint.

Download the AEM™ package for NEBA {{site.nebaVersion}}

Installation in Apache Sling

We recommend obtaining the AEM package ZIP file and installing the bundles contained therein with a method of choice, e.g. using the Maven Sling Plugin.

Using MAVEN

NEBA is built using Maven. Core, API and a delivery package containing NEBA and it's runtime dependencies are available from maven central. For developing with NEBA, including the API in your project is all you need to do:

{% highlight xml %} io.neba io.neba.neba-api {{site.nebaVersion}} {% endhighlight %}

If you'd also like to browse or debug the core, add it to your project's dependencies like so:

{% highlight xml %} io.neba io.neba.neba-core {{site.nebaVersion}} {% endhighlight %}

Finally, you can reference the deployment package like so:

{% highlight xml %} io.neba io.neba.neba-delivery {{site.nebaVersion}} zip {% endhighlight %}

For instance in order to install it with the Content Package Maven Plugin:

{% highlight xml %} com.day.jcr.vault content-package-maven-plugin 0.0.20 io.neba io.neba.neba-delivery zip http://localhost:4502/crx/packmgr/service.jsp {% endhighlight %}

You may also want to configure the following repository for the eclipse artifacts NEBA depends on:

{% highlight xml %} eclipse-gemini Eclipse Gemini Maven repository http://download.eclipse.org/gemini/mvn/ true false {% endhighlight %}
